Pre-flight checklist

Boot the Artix Linux live ISO (dinit edition). Confirm you have internet — ethernet is easiest on the live environment; if you need Wi-Fi see Step 11 for iwd usage.

verify boot mode (must show UEFI files)
ls /sys/firmware/efi/efivars
⚠
If the directory does not exist, your system booted in BIOS/Legacy mode. Reboot and force UEFI in your firmware settings. This guide requires UEFI.
confirm target disk
lsblk -o NAME,SIZE,TYPE,MOUNTPOINT
# expect to see nvme0n1 — your target disk
verify internet
ping -c 3 artixlinux.org
sync clock
dinitctl start ntpd
# or
rdate -n time.cloudflare.com
1
Partition the disk

We create a GPT table with two partitions: a small EFI System Partition and a large partition that will hold the LUKS container.

DeviceSizeTypePurpose
nvme0n1p1 512 MiB EFI ESP — GRUB EFI files, unencrypted
nvme0n1p2 Remainder LUKS2 Encrypted container → ext4 root + swapfile
gdisk — interactive partitioner
gdisk /dev/nvme0n1

# Inside gdisk:
o          # create new GPT table, confirm with y
n          # new partition
             # partition number: 1 (default)
             # first sector: (default)
             # last sector: +512M
             # type code: ef00  ← EFI System Partition
n          # new partition
             # partition number: 2 (default)
             # first sector: (default)
             # last sector: (default — use rest of disk)
             # type code: 8309  ← Linux LUKS
w          # write and exit
verify
lsblk /dev/nvme0n1
# should show nvme0n1p1 (512M) and nvme0n1p2
2
Create & open the LUKS2 container

We use LUKS2 with Argon2id KDF (the modern default) and aes-xts-plain64 cipher. This provides strong encryption resistant to brute-force even with a short passphrase.

format partition 2 as LUKS2
cryptsetup luksFormat \
  --type      luks2 \
  --cipher    aes-xts-plain64 \
  --key-size  512 \
  --hash      sha512 \
  --pbkdf     argon2id \
  --iter-time 3000 \
  /dev/nvme0n1p2

# Type YES in uppercase when prompted, then enter your passphrase twice
⚠
Passphrase: Use something long and memorable. There is no recovery if you forget it without a backup key. Consider adding a keyfile later (cryptsetup luksAddKey).
open (map) the container
cryptsetup open /dev/nvme0n1p2 cryptroot
# Enter your passphrase
# The decrypted device is now at /dev/mapper/cryptroot
verify mapped device
ls -la /dev/mapper/cryptroot
3
Format partitions & mount
format EFI partition as FAT32
mkfs.fat -F32 -n EFI /dev/nvme0n1p1
format LUKS inner device as ext4
mkfs.ext4 -L ROOT /dev/mapper/cryptroot
mount everything
# Mount root first
mount /dev/mapper/cryptroot /mnt

# Create and mount EFI
mkdir -p /mnt/boot/efi
mount /dev/nvme0n1p1 /mnt/boot/efi

Optional: create a swapfile inside the LUKS container

A swapfile inside the encrypted container is simpler than a swap partition — it gets encrypted automatically.

create 8 GiB swapfile (adjust to ~RAM size)
dd if=/dev/zero of=/mnt/swapfile bs=1M count=8192 status=progress
chmod 600 /mnt/swapfile
mkswap /mnt/swapfile
swapon /mnt/swapfile
4
Install base system

Use basestrap (Artix's equivalent of pacstrap) to install the essential packages into /mnt. We install the dinit variants of all init-dependent packages.

install base + dinit + kernel + firmware
basestrap /mnt \
  base base-devel \
  linux linux-headers linux-firmware \
  dinit dinit-rc \
  cryptsetup lvm2 \
  grub efibootmgr \
  vim neovim \
  iwd iwd-dinit \
  man-db man-pages \
  curl wget git
ℹ
dinit-rc provides the OpenRC-compatible service scripts and the /etc/rc.conf configuration layer that dinit uses on Artix. It is required alongside dinit itself.
✦
If you are on Wi-Fi during the live environment and need to connect first, see Step 11 for iwctl usage — the tool is available on the live ISO.

8
Configure & generate initramfs

This is the most critical step for LUKS. We must add encrypt to the HOOKS array in mkinitcpio.conf so the initramfs can unlock the LUKS container at boot.

edit /etc/mkinitcpio.conf
nvim /etc/mkinitcpio.conf

# Find the HOOKS line and set it to exactly this:
HOOKS=(base udev autodetect microcode modconf kms keyboard keymap block encrypt filesystems fsck)

# Important notes:
#  - 'keyboard' and 'keymap' must be BEFORE 'encrypt'
#    so you can type your passphrase at boot
#  - 'encrypt' must be BEFORE 'filesystems'
#  - 'microcode' handles AMD/Intel microcode loading
ℹ
If you have an Intel CPU, install intel-ucode. For AMD, install amd-ucode. The microcode hook will pick up whichever is present automatically.
pacman -S intel-ucode or pacman -S amd-ucode
regenerate the initramfs
mkinitcpio -P
# -P regenerates for all installed kernels
# Look for: ==> Image generation successful
9
Install & configure GRUB

GRUB needs to know where the LUKS partition is so it can pass cryptdevice to the kernel on boot. We need the UUID of the raw LUKS partition (not the mapped device).

get UUID of the LUKS partition
blkid /dev/nvme0n1p2
# Copy the UUID value — looks like: a1b2c3d4-e5f6-7890-abcd-ef1234567890
# You can also run: blkid -s UUID -o value /dev/nvme0n1p2
edit /etc/default/grub
nvim /etc/default/grub

# Find and edit the GRUB_CMDLINE_LINUX_DEFAULT line:
# Replace YOUR-UUID with the UUID from blkid above

GRUB_CMDLINE_LINUX_DEFAULT="loglevel=3 quiet cryptdevice=UUID=YOUR-UUID:cryptroot root=/dev/mapper/cryptroot"

# Also set (or verify):
GRUB_ENABLE_CRYPTODISK=y

# Optional: show GRUB menu (remove 'quiet' if you want verbose boot)
✕
GRUB_ENABLE_CRYPTODISK=y is mandatory. Without it, GRUB cannot read the encrypted partition to find its configuration and will fail to boot.
install GRUB to EFI
grub-install \
  --target=x86_64-efi \
  --efi-directory=/boot/efi \
  --bootloader-id=ARTIX \
  --recheck

# Expected: Installation finished. No error reported.
generate GRUB config
grub-mkconfig -o /boot/grub/grub.cfg

# You should see "Found linux image" and "Found initrd image" lines
# Verify the cryptdevice parameter appears in the output
ℹ
With GRUB_ENABLE_CRYPTODISK=y, GRUB will ask for your LUKS passphrase twice on boot: once for GRUB to access its own config, and once from the initramfs. This is normal. You can eliminate the double prompt by embedding a keyfile, which is an optional post-install hardening step.

iwd networking (Wi-Fi)

Artix uses iwd (iNet wireless daemon) for Wi-Fi. It can also handle IP assignment via its built-in DHCP client, so no separate dhcpcd is required for simple setups.

Enable iwd DHCP client

/etc/iwd/main.conf
mkdir -p /etc/iwd
nvim /etc/iwd/main.conf

# Add these contents:
[General]
EnableNetworkConfiguration=true

[Network]
NameResolvingService=resolvconf

Enable the iwd dinit service

enable iwd at boot
ln -s /etc/dinit.d/iwd /etc/dinit.d/boot.d/iwd

Connect to Wi-Fi at boot / live environment

iwctl — interactive Wi-Fi setup
iwctl

# Inside iwctl interactive shell:
[iwd]# device list
[iwd]# station wlan0 scan
[iwd]# station wlan0 get-networks
[iwd]# station wlan0 connect "MyNetwork"
         # enter passphrase when prompted
[iwd]# quit
✦
iwd remembers networks you connect to. Credentials are stored in /var/lib/iwd/ and reconnection is automatic on subsequent boots.

Optional: install resolvconf for DNS

openresolv for /etc/resolv.conf management
pacman -S openresolv

CUPS printing with Avahi discovery

Artix does not ship cups-browsed because it is systemd-dependent. Printer auto-discovery is instead handled by Avahi (mDNS/DNS-SD / Bonjour), which works perfectly with dinit and provides the same discovery capability for network printers.

Install packages

cups + avahi + drivers
pacman -S \
  cups cups-dinit \
  avahi avahi-dinit \
  nss-mdns \
  print-manager \
  ghostscript gsfonts \
  foomatic-db foomatic-db-engine \
  hplip \                      # HP printers — remove if not needed
  system-config-printer        # GUI printer manager (optional)
ℹ
avahi-dinit is the Artix dinit service unit for avahi-daemon. The nss-mdns package enables .local hostname resolution (needed so CUPS can find Avahi-discovered printers by hostname).

Configure nss-mdns for .local resolution

/etc/nsswitch.conf — add mdns4_minimal
nvim /etc/nsswitch.conf

# Find the 'hosts:' line and make it look like this:
hosts: mymachines mdns4_minimal [NOTFOUND=return] resolve [!UNAVAIL=return] files myhostname dns

# The key addition is 'mdns4_minimal [NOTFOUND=return]' before 'resolve'

Configure Avahi to browse for printers

/etc/avahi/avahi-daemon.conf — verify these settings
nvim /etc/avahi/avahi-daemon.conf

# Ensure these are set under [server]:
use-ipv4=yes
use-ipv6=yes

# And under [publish]:
publish-hinfo=no
publish-workstation=no

Enable CUPS Avahi backend

/etc/cups/cups-files.conf — allow network browsing
nvim /etc/cups/cups-files.conf
# Ensure (or add):
Browsing Yes
BrowseLocalProtocols dnssd
/etc/cups/cupsd.conf — allow localhost access
nvim /etc/cups/cupsd.conf
# Verify / add near the top:
Listen localhost:631
Listen /run/cups/cups.sock

# For web UI access from localhost:
<Location />
  Order allow,deny
  Allow localhost
</Location>

<Location /admin>
  Order allow,deny
  Allow localhost
</Location>

Enable services

enable cups and avahi via dinit boot.d symlinks
ln -s /etc/dinit.d/cupsd   /etc/dinit.d/boot.d/cupsd
ln -s /etc/dinit.d/avahi-daemon /etc/dinit.d/boot.d/avahi-daemon
✓
How discovery works without cups-browsed: Avahi-daemon listens on the network for DNS-SD/mDNS announcements from printers advertising IPP (Internet Printing Protocol). When you open the CUPS web UI at http://localhost:631 or use system-config-printer, Avahi-discovered printers appear automatically as available network printers — no cups-browsed needed.

Add user to lp group

allow user to manage printers (replace 'alice')
usermod -aG lp,sys alice
14
Exit chroot & reboot
exit chroot and unmount everything cleanly
exit                          # leave chroot
umount -R /mnt                 # unmount all
cryptsetup close cryptroot    # close LUKS
reboot
ℹ
On first boot: GRUB will prompt for your LUKS passphrase (once for GRUB itself, then once for the initramfs unlock). This is expected. After unlocking, dinit starts your services.
